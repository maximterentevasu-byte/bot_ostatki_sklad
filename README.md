# Telegram bot: Excel upload + barcode check

Бот на **Node.js + Telegraf** для запуска на **Railway**.

## Что изменено

- улучшено распознавание штрихкода по фото;
- бот остается в режиме сканирования после успешной проверки и после ошибок распознавания;
- добавлена кнопка **«Ввести ШК вручную»**;
- после ручного ввода бот снова готов принимать следующее фото без повторного нажатия **«Проверить ШК»**.

## Что умеет

### 1. Кнопка «Загрузить файл»
- принимает Excel-файл в Telegram;
- удаляет старый Excel из папки `GITHUB_UPLOAD_DIR` в GitHub;
- загружает новый Excel;
- в папке всегда остается только один актуальный файл.

### 2. Кнопка «Проверить ШК»
- переводит бота в режим непрерывного сканирования;
- бот принимает фото штрихкода одно за другим без повторного нажатия кнопки;
- распознает **EAN-13**;
- при неудаче предлагает отправить новое фото или использовать ручной ввод;
- ищет штрихкод в Excel и возвращает информацию по товару.

### 3. Кнопка «Ввести ШК вручную»
- позволяет ввести код вручную, если фото не распозналось;
- принимает только валидный **EAN-13** из 13 цифр;
- после обработки сохраняет режим дальнейшей проверки.

## Как улучшено распознавание

Теперь бот пробует несколько вариантов обработки изображения перед распознаванием:
- оттенки серого;
- нормализацию контраста;
- усиление контраста;
- инверсию;
- легкое размытие;
- увеличение изображения;
- поворот на 90° и 270°;
- распознавание по центральным и обрезанным фрагментам кадра;
- распознавание по горизонтальным полосам изображения.

Это заметно повышает шанс чтения ШК с фото, где код снят неидеально, под углом или при плохом освещении.

## Установка локально

```bash
npm install
npm start
```

## Переменные окружения

Создайте `.env` на основе `.env.example`:

```env
BOT_TOKEN=1234567890:AAExampleYourTelegramBotToken
GITHUB_TOKEN=github_pat_xxxxxxxxxxxxxxxxx
GITHUB_OWNER=your-github-login-or-org
GITHUB_REPO=your-repo-name
GITHUB_BRANCH=main
GITHUB_UPLOAD_DIR=uploads
```

## Деплой на Railway

1. Загрузите проект в GitHub.
2. В Railway создайте проект из GitHub-репозитория.
3. В Railway → **Variables** добавьте все переменные окружения.
4. Railway сам установит зависимости и выполнит:

```bash
npm start
```

## Логика Excel

По найденной строке в Excel бот возвращает:
- `Название товара` → столбец **C**
- `Общий остаток товара` → столбец **G**
- `Общий в т.ч. в АКЦИИ` → столбец **F**
- `Остаток склад` → столбец **H**
- `Каменская ост` → столбец **I**
- `Победы ост` → столбец **K**
- `Асбест ост` → столбец **J**
- `Все продажи за 14 дней` → столбец **L**

Штрихкод ищется в столбце **D**.

## Важно

- В репозиторий не загружайте реальный `.env`.
- Храните реальные токены только в **Railway Variables**.
- Для распознавания лучше присылать четкое фото с хорошим светом.
- Если код не распознался с фото, используйте кнопку **«Ввести ШК вручную»**.
